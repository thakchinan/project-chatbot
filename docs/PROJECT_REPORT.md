# รายงานโครงงานนวัตกรรม SmartBrain Care

---

## 1. ที่มาและแนวคิดของการสร้างนวัตกรรม

ในปัจจุบันสังคมไทยต้องเผชิญกับปัญหาความเครียดและภาวะซึมเศร้าเพิ่มขึ้นอย่างต่อเนื่อง อันเนื่องมาจากการแข่งขันที่สูงขึ้นในทุกมิติของการดำเนินชีวิต ทั้งด้านการทำงาน การศึกษา และความไม่มั่นคงทางเศรษฐกิจ บุคคลจำนวนมากต้องปฏิบัติหน้าที่แข่งกับเวลาภายใต้แรงกดดันจากสภาพสังคมและเทคโนโลยีที่เปลี่ยนแปลงอย่างรวดเร็ว ส่งผลให้เกิดความเครียดสะสมโดยไม่รู้ตัว ซึ่งหากไม่ได้รับการดูแลและจัดการอย่างเหมาะสม ความเครียดเหล่านี้อาจพัฒนาไปสู่ภาวะซึมเศร้า ซึ่งเป็นปัญหาสุขภาพจิตระดับชาติได้

ปัญหาความเครียดและภาวะซึมเศร้าส่งผลกระทบโดยตรงต่อสุขภาพทั้งทางร่างกายและจิตใจ อาทิ ภาวะนอนไม่หลับ ความดันโลหิตสูง ระบบภูมิคุ้มกันอ่อนแอ และประสิทธิภาพการทำงานที่ลดลง นอกจากนี้ สภาวะทางอารมณ์ยังส่งผลโดยตรงต่อคลื่นสมอง (Brainwaves) ซึ่งเป็นสัญญาณชีวภาพที่สะท้อนการทำงานของสมอง โดยเฉพาะในผู้ที่มีภาวะซึมเศร้าหรือเครียดจัด มักตรวจพบรูปแบบความผิดปกติของคลื่นสมองในบางย่านความถี่ที่มีความสัมพันธ์กับการเปลี่ยนแปลงของอารมณ์ ความคิด และพฤติกรรม

อย่างไรก็ตาม กระบวนการคัดกรองหรือตรวจวัดภาวะเครียดและซึมเศร้าในปัจจุบัน ส่วนใหญ่ยังคงพึ่งพาการสังเกตพฤติกรรมหรือการทำแบบสอบถาม (Questionnaire) ซึ่งเป็นข้อมูลเชิงอัตนัย (Subjective) ที่อาจไม่สะท้อนสภาพความเป็นจริงของร่างกายได้อย่างแม่นยำ อีกทั้งผู้ที่มีอาการจำนวนมากมักไม่ตระหนักถึงปัญหาของตนเองจนกระทั่งส่งผลกระทบในระดับรุนแรง จึงมีความจำเป็นอย่างยิ่งที่จะต้องมีเครื่องมือที่สามารถช่วยตรวจวัดและวิเคราะห์สภาวะทางอารมณ์ได้แบบเรียลไทม์ผ่านสัญญาณทางสรีรวิทยา

แม้ว่าปัจจุบันจะมีเทคโนโลยีการตรวจวัดคลื่นสมอง (EEG) อยู่แล้ว แต่รูปแบบการใช้งานทั่วไปยังคงเป็นอุปสรรคสำคัญ (Pain Points) โดยเฉพาะอย่างยิ่งสำหรับกลุ่มเป้าหมายที่เป็นผู้สูงอายุ กล่าวคือ เครื่องมือวัดคลื่นสมองทั่วไปมักแสดงผลในรูปแบบของกราฟสัญญาณดิบหรือแผนภูมิทางสถิติผ่านหน้าจอคอมพิวเตอร์หรือแอปพลิเคชัน ซึ่งมีความซับซ้อนและเข้าใจยากสำหรับบุคคลทั่วไป จำเป็นต้องอาศัยผู้เชี่ยวชาญในการแปลผล อีกทั้งแอปพลิเคชันสุขภาพจิตที่มีอยู่ในตลาดมักบังคับให้ผู้ใช้งานต้องเพ่งมองหน้าจอเพื่ออ่านบทความขนาดยาว หรือต้องใช้ทักษะความแม่นยำของนิ้วมือในการพิมพ์ข้อความเพื่อโต้ตอบ ซึ่งไม่สอดคล้องกับสรีระของผู้สูงอายุที่มักมีข้อจำกัดด้านสายตาและความถนัดในการใช้สมาร์ตโฟน ยิ่งไปกว่านั้น การนำเสนอผลลัพธ์ที่เป็นเพียงกราฟหรือข้อความแจ้งเตือนอัตโนมัติ ยังขาดมิติของความเข้าอกเข้าใจ (Empathy) ซึ่งเป็นสิ่งที่ผู้ที่มีภาวะความเครียดต้องการมากที่สุด

จากปัญหาและข้อจำกัดดังกล่าว จึงเกิดเป็นแนวคิดในการพัฒนา **SmartBrain Care** ซึ่งเป็นนวัตกรรมที่มุ่งแก้ปัญหาเหล่านี้อย่างตรงจุด ด้วยการยกระดับจากการเป็นเพียงเครื่องมือแสดงกราฟคลื่นสมอง มาสู่การเป็น **ผู้ช่วยดูแลสุขภาพใจอัจฉริยะ** โดยโครงงานนี้ได้นำเทคโนโลยีตัวแทนสนทนาปัญญาประดิษฐ์ (AI Chatbot) ที่ทำงานร่วมกับเทคนิค RAG (Retrieval-Augmented Generation) และระบบสั่งงานด้วยเสียง (Voice Interaction) เข้ามาเป็นหัวใจหลักของระบบ เพื่อให้ผู้ใช้งานสามารถพูดคุยกับระบบได้ด้วยเสียงภาษาไทยเสมือนสนทนากับคนดูแลส่วนตัว ตัดปัญหาการต้องเพ่งมองจอหรือพิมพ์ข้อความ ทำให้ผู้สูงอายุเข้าถึงบริการได้ง่ายขึ้น (Accessibility) โดยระบบจะนำผลการวิเคราะห์คลื่นสมองส่วนบุคคล (Personalized EEG Data) มาประมวลผลร่วมกับฐานข้อมูลทางการแพทย์ เพื่อให้คำแนะนำในการผ่อนคลายความเครียดที่ถูกต้อง แม่นยำ และมีความนุ่มนวล เข้าอกเข้าใจ สอดคล้องกับสภาวะอารมณ์ของผู้ใช้งานในขณะนั้นมากที่สุด

## 2. วัตถุประสงค์

1. เพื่อพัฒนาเครื่องมือคัดกรองความเสี่ยงที่สามารถประเมินระดับความเครียดและความเสี่ยงภาวะซึมเศร้าได้อย่างง่าย สะดวก และมีความแม่นยำ

2. เพื่อเชื่อมโยงข้อมูลคลื่นสมอง (EEG) เข้ากับการประเมินภาวะความเครียดและความเสี่ยงต่อภาวะซึมเศร้า เพื่อเพิ่มความถูกต้องและความน่าเชื่อถือของผลประเมิน

3. เพื่อประยุกต์ใช้เทคโนโลยีปัญญาประดิษฐ์ (AI) ในการวิเคราะห์และทำนายความเสี่ยงของภาวะซึมเศร้าเบื้องต้นอย่างมีประสิทธิภาพ

4. เพื่อพัฒนาตัวแทนสนทนาปัญญาประดิษฐ์ (AI Chatbot) ที่รองรับการสั่งงานด้วยเสียงและการโต้ตอบแบบมีความเห็นอกเห็นใจ (Empathetic) สำหรับผู้สูงอายุ โดยใช้เทคนิค RAG (Retrieval-Augmented Generation) ในการให้คำแนะนำสุขภาพจิตที่แม่นยำและเชื่อมโยงกับผลตรวจคลื่นสมองส่วนบุคคล

## 4. การทบทวนวรรณกรรม (Literature Review)

### 4.1 ภาวะซึมเศร้าและการคัดกรอง

แบบสอบถาม PHQ-9 (Patient Health Questionnaire-9) พัฒนาโดย Drs. Kroenke, Spitzer และ Williams (2001) เป็นเครื่องมือคัดกรองภาวะซึมเศร้ามาตรฐานสากลที่ได้รับการรับรองจาก WHO และ APA ประกอบด้วย 9 คำถาม คะแนนรวม 0-27 แบ่งเป็น 5 ระดับ ตั้งแต่ปกติจนถึงรุนแรง อย่างไรก็ตาม PHQ-9 เป็นข้อมูลเชิงอัตนัย (Subjective) ที่ขึ้นอยู่กับการรายงานตนเองของผู้ป่วย จึงอาจมีข้อจำกัดด้านความแม่นยำ

### 4.2 คลื่นไฟฟ้าสมอง (EEG) กับสุขภาพจิต

คลื่นสมองแบ่งเป็น 5 ประเภทหลัก: Delta (0.5-4 Hz) สัมพันธ์กับการนอนหลับลึก, Theta (4-8 Hz) สมาธิลึกและผ่อนคลาย, Alpha (8-13 Hz) ตื่นตัวอย่างสงบ, Beta (13-30 Hz) คิดวิเคราะห์และทำงาน, Gamma (30+ Hz) สมาธิสูง งานวิจัยของ Krigolson et al. (2017) แสดงว่าอุปกรณ์ EEG สำหรับผู้บริโภคอย่าง Muse สามารถวัดคลื่นสมองได้อย่างน่าเชื่อถือในระดับหนึ่ง โดยเฉพาะคลื่น Alpha ที่สัมพันธ์กับความผ่อนคลาย

### 4.3 Retrieval-Augmented Generation (RAG)

Lewis et al. (2020) นำเสนอเทคนิค RAG ที่ผสานการสืบค้นข้อมูล (Retrieval) เข้ากับโมเดลภาษาขนาดใหญ่ (LLM) เพื่อลดปัญหา Hallucination และเพิ่มความถูกต้องของคำตอบ โดยระบบจะสืบค้นข้อมูลจากฐานความรู้ที่เชื่อถือได้ก่อนสร้างคำตอบ

### 4.4 การสืบค้นสิทธิบัตร (Patent Search)

จากการสืบค้นสิทธิบัตรที่เกี่ยวข้อง พบว่ามีสิทธิบัตรด้านอุปกรณ์ EEG และการวิเคราะห์อารมณ์จากคลื่นสมองอยู่จำนวนหนึ่ง แต่ยังไม่พบสิทธิบัตรที่ผสานการวิเคราะห์ EEG เข้ากับ AI Chatbot ที่ใช้เทคนิค RAG และระบบเสียงภาษาไทยสำหรับผู้สูงอายุโดยเฉพาะ ซึ่งเป็นจุดแตกต่างหลักของนวัตกรรม SmartBrain Care

## 5. คุณสมบัติ/คุณลักษณะเฉพาะและขอบเขตการใช้งาน

### 5.1 คุณสมบัติ / คุณลักษณะเฉพาะ

**1) ระบบประมวลผลสัญญาณคลื่นสมองแบบเรียลไทม์ (Real-time EEG Processing)**

ระบบเชื่อมต่อกับอุปกรณ์ Muse S ผ่านสัญญาณ Bluetooth (BLE) รับข้อมูลสัญญาณ EEG ดิบ แปลงเป็นค่า μV ผ่านกระบวนการ Parsing (12-bit unsigned, reference 1682.815 μV) สะสมข้อมูล 256 samples แล้วประมวลผลด้วย Fast Fourier Transform (FFT) เพื่อแยก Band Powers ทั้ง 5 ย่านความถี่ (Delta, Theta, Alpha, Beta, Gamma) จากนั้นส่งข้อมูลไปยังโมเดล Deep Learning (TSception) ผ่าน Emotion Detection API เพื่อจำแนกสภาวะอารมณ์ได้ทันที

**สถาปัตยกรรมที่พัฒนาแล้ว:**
- `MuseService` — เชื่อมต่อ BLE, รับ/แปลง EEG packets, คำนวณ FFT (449 บรรทัด)
- `FFTCalculator` — คำนวณ Fast Fourier Transform แยก band powers
- `EmotionDetectionService` — เชื่อมต่อ Emotion API สำหรับจำแนกอารมณ์
- `Emotion API (FastAPI/Python)` — โหลด TSception model + K-Means scaler + Rule-based fallback (447 บรรทัด)

**2) ตัวแทนสนทนาสั่งงานด้วยเสียงภาษาไทย (Voice-First AI Chatbot)**

ออกแบบเพื่อผู้สูงอายุโดยเฉพาะ รองรับการรับคำสั่งเสียงภาษาไทยผ่าน OpenAI Whisper API (Speech-to-Text) และสังเคราะห์เสียงพูดตอบกลับผ่าน Flutter TTS (Text-to-Speech) ที่เป็นธรรมชาติ มีระบบ Auto-speak ที่ AI จะพูดตอบกลับอัตโนมัติเมื่อเปิดใช้งาน

**สถาปัตยกรรมที่พัฒนาแล้ว:**
- `STTService` — บันทึกเสียงผ่าน AudioRecorder แล้วส่งไป Whisper API แปลงเป็นข้อความ (241 บรรทัด)
- `TTSService` — FlutterTTS ภาษาไทย (th-TH) ความเร็ว 0.5 (82 บรรทัด)
- `RecommendationScreen` — UI แชท พร้อมปุ่มไมค์, animation pulse, voice listening bar (793 บรรทัด)

**3) การสนทนาที่ปรับตามสภาวะอารมณ์ (Emotion-Adaptive Response)**

ระบบผสานข้อมูลคลื่นสมองเข้ากับ AI โดย ChatGPTService มีฟังก์ชัน `sendMessageWithBrainwaveContext()` ที่นำค่าคลื่นสมอง 5 ประเภท + Attention/Meditation scores รวมเข้าไปใน prompt เพื่อให้ AI รับรู้สถานะอารมณ์จริงของผู้ใช้และปรับโทนการตอบให้เหมาะสม

**4) การประมวลผลคำแนะนำทางการแพทย์ด้วย RAG (Retrieval-Augmented Generation)**

ระบบเชื่อมต่อ GPT-4o-mini เข้ากับ Vector Database (Supabase pgvector) ที่จัดเก็บฐานความรู้ 13+ entries จากกรมสุขภาพจิต ใช้ Hybrid Search (Vector 70% + Keyword 30%) ในการสืบค้น มี 3 ระดับการค้นหา: Keyword Search → Vector Search → Broad Search

**สถาปัตยกรรมที่พัฒนาแล้ว:**
- `RAGService` — Embedding (text-embedding-3-small, 1536 dims), Vector/Keyword/Broad search, Context building (485 บรรทัด)
- `ChatGPTService` — GPT-4o-mini + RAG integration + Brainwave context (311 บรรทัด)
- Database Functions: `search_knowledge()`, `hybrid_search()`, `get_user_context()`

**5) ระบบเฝ้าระวังความปลอดภัย (Safety Guardrail)**

ในคำถามข้อ 9 ของ PHQ-9 (ความคิดทำร้ายตนเอง) ระบบแสดงข้อมูลติดต่อฉุกเฉินทันที ได้แก่ สายด่วนสุขภาพจิต 1323 และสมาคมสะมาริตันส์ นอกจากนี้ RAG Knowledge Base มี entries เกี่ยวกับเบอร์ฉุกเฉินที่ AI สามารถแนะนำได้

### 5.2 ขอบเขตการใช้งาน

**กลุ่มผู้ใช้งานเป้าหมาย:**
- **ผู้สูงอายุ** — กลุ่มผู้ใช้งานหลักในการโต้ตอบและรับการประเมิน
- **ผู้ดูแล (Caregiver)** — สามารถติดตามสรุปภาพรวมผ่าน Health Dashboard

**รูปแบบการทำงาน:** ใช้งานผ่านแอปพลิเคชัน Flutter บนสมาร์ตโฟน (iOS/Android) เชื่อมต่อ Muse S EEG Headset ผ่าน BLE จำเป็นต้องมีอินเทอร์เน็ตเพื่อส่งข้อมูลไปประมวลผลผ่าน Backend (Supabase Cloud + OpenAI API)

**ข้อจำกัดทางคลินิก:** เป็น "เครื่องมือคัดกรองเบื้องต้น" สำหรับเฝ้าระวังและให้คำปรึกษาเพื่อลดความเครียดเท่านั้น ไม่สามารถใช้วินิจฉัยโรคซึมเศร้าหรือสั่งการรักษาแทนแพทย์ได้

## 6. จุดเด่นที่แตกต่างจากผลงานอื่น

### 6.1 การผสานข้อมูลคลื่นสมองเข้ากับบริบทของ AI (Physiological-AI Integration)

**สิ่งที่แตกต่าง:** อุปกรณ์ EEG ทั่วไปนำเสนอผลลัพธ์เป็นกราฟสัญญาณดิบที่เข้าใจยาก

**กลไกของ SmartBrain Care:** นำข้อมูล EEG เรียลไทม์ประมวลผลผ่าน Deep Learning (TSception + Rule-based fallback) จำแนกอารมณ์ 10 ประเภท (neutral, happy, sad, angry, fearful, calm, stressed, anxious, surprised, disgusted) แล้วส่งเป็นข้อมูลบริบทให้ AI Chatbot ผ่าน `sendMessageWithBrainwaveContext()`

### 6.2 การตอบโต้ที่เข้าอกเข้าใจ (Emotion-Adaptive & Empathetic Response)

**สิ่งที่แตกต่าง:** แชทบอททั่วไปตอบแบบอัตโนมัติตามชุดคำสั่ง ขาดความเข้าอกเข้าใจ

**กลไก:** ระบบวิเคราะห์ดัชนีความเครียดจากคลื่นสมอง + Sentiment จากบทสนทนา ปรับ System Prompt ให้ AI ตอบสนองด้วยโทนที่นุ่มนวลเหมือนคนดูแลส่วนตัว ใช้ภาษาไทยที่เป็นกันเอง ไม่ใช้ emoji หรือ markdown

### 6.3 การขจัดปัญหา AI ให้ข้อมูลเท็จด้วย RAG (RAG-Powered Medical Accuracy)

**สิ่งที่แตกต่าง:** LLM ทั่วไปมีความเสี่ยง Hallucination สูงในบริบทการแพทย์

**กลไก:** ใช้ RAG ดึงข้อมูลจาก Vector Database (pgvector) ที่อ้างอิงคู่มือสุขภาพจิต System Prompt บังคับให้ AI ใช้เฉพาะข้อมูลอ้างอิงที่ให้มา หากไม่มีข้อมูลให้ตอบว่าไม่มี รวมถึงมีระบบแจ้งเตือนฉุกเฉิน (สายด่วน 1323, สมาคมสะมาริตันส์)

### 6.4 การออกแบบที่ไร้รอยต่อเพื่อผู้สูงอายุ (Voice-First & Elderly-Centric Design)

**สิ่งที่แตกต่าง:** แอปสุขภาพจิตทั่วไปบังคับให้เพ่งจอหรือพิมพ์ข้อความ

**กลไก:** ขับเคลื่อนด้วยเสียงภาษาไทย 100% ผ่าน OpenAI Whisper (STT) + Flutter TTS (TTS) มี Auto-speak ให้ AI พูดตอบกลับอัตโนมัติ ปุ่มไมค์ขนาดใหญ่พร้อม Pulse Animation แสดงสถานะกำลังฟัง ตัวอักษรขนาดใหญ่อ่านง่าย

## 7. หลักการ วิธีการ และขั้นตอนการทำงาน

### 7.1 หลักการของนวัตกรรม (Principles)

SmartBrain Care ทำงานบนหลักการบูรณาการเทคโนโลยี 3 ส่วน:

**1) Brain-Computer Interface (BCI):** ใช้ Muse S EEG Headset รับสัญญาณคลื่นไฟฟ้าสมอง 4 channels (TP9, AF7, AF8, TP10) ผ่าน BLE ประมวลผลด้วย FFT แยก 5 band powers

**2) Analytical AI:** ใช้โมเดล TSception (Deep Learning) + K-Means Scaler/Encoder จำแนกสภาวะอารมณ์ 10 ประเภท พร้อม Rule-based fallback สำหรับกรณีที่โมเดลไม่พร้อม

**3) Generative AI:** ใช้ GPT-4o-mini ทำงานร่วมกับ RAG (pgvector + Hybrid Search) และ Voice I/O (Whisper + Flutter TTS) สร้างตัวแทนสนทนาที่ให้คำปรึกษาถูกต้องตามหลักการแพทย์

### 7.3 ขั้นตอนการทำงาน (Working Process)

#### ขั้นตอนที่ 1: การรับสัญญาณคลื่นสมอง (EEG Acquisition)

1. ผู้ใช้สวมอุปกรณ์ Muse S Headband
2. แอปสแกนอุปกรณ์ BLE ที่มีชื่อ "Muse"
3. เชื่อมต่อ GATT แล้วค้นหา EEG Characteristics (UUID: 273e0003-0009)
4. รับ raw bytes → แปลงเป็น μV samples (12-bit unsigned)
5. สะสม 256 samples ใน sliding window

#### ขั้นตอนที่ 2: การประมวลผลคลื่นสมอง (EEG Processing)

1. คำนวณ FFT จาก window 256 samples
2. แยก Band Powers: Delta, Theta, Alpha, Beta, Gamma
3. Normalize เป็นเปอร์เซ็นต์ (ผลรวม = 100%)
4. อัปเดต UI แบบเรียลไทม์ผ่าน ChangeNotifier

#### ขั้นตอนที่ 3: การตรวจจับอารมณ์ (Emotion Detection)

1. ทุก 5 วินาที ดึงข้อมูล EEG ล่าสุด
2. ส่งไปยัง Emotion API (FastAPI Server)
3. TSception Model วิเคราะห์ → จำแนกอารมณ์ 10 ประเภท
4. Rule-based fallback หากโมเดลไม่พร้อม
5. แสดงผลบน Emotion Detection Card พร้อม Confidence Score

#### ขั้นตอนที่ 4: การสนทนากับ AI (AI Chatbot with RAG)

1. ผู้ใช้พูด (STT) หรือพิมพ์คำถาม
2. Whisper API แปลงเสียง → ข้อความภาษาไทย
3. RAG ค้นหาความรู้ที่เกี่ยวข้องจาก Knowledge Base:
   - Keyword Search (ค้นหาคำสำคัญ + synonyms)
   - Vector Search (cosine similarity, threshold 0.7)
   - Broad Search (fallback แยกคำค้นหา)
4. สร้าง Enhanced System Prompt = Base Prompt + RAG Context + User Context (ข้อมูลคลื่นสมอง + ระดับเครียด + กิจกรรมล่าสุด)
5. GPT-4o-mini สร้างคำตอบที่ปรับตามอารมณ์
6. TTS อ่านคำตอบเป็นเสียงภาษาไทย (Auto-speak)

#### ขั้นตอนที่ 5: การประเมิน PHQ-9 (Depression Screening)

1. ผู้ใช้ทำแบบทดสอบ 9 ข้อ (0-3 คะแนนต่อข้อ)
2. คำนวณคะแนนรวม (0-27) → จำแนก 5 ระดับ
3. แสดง Result Sheet พร้อม gauge + คำแนะนำ
4. บันทึกผลลง Supabase (test_results table)
5. Safety Guardrail: ข้อ 9 แสดงเบอร์ฉุกเฉิน

#### ขั้นตอนที่ 6: การจัดเก็บและติดตามข้อมูล (Data Storage & Monitoring)

1. ข้อมูลทุกประเภทจัดเก็บใน Supabase Cloud (10 ตาราง)
2. Row Level Security (RLS) ปกป้องข้อมูลส่วนบุคคล
3. Realtime Subscription อัปเดต UI อัตโนมัติเมื่อมีข้อมูลใหม่
4. Health Dashboard สรุปภาพรวมสุขภาพสมอง
